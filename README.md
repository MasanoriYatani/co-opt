# Optical System Visualizer

## Overview
The Optical System Visualizer is a web-based application designed to visualize optical systems based on lens design data. Users can manage lens parameters through a table interface and see graphical representations of the optical components and their relationships.

## Project Structure
```
optical-system-visualizer
├── src
│   ├── index.html        # Main HTML document
│   ├── main.js           # Entry point of the application
│   ├── lensTable.js      # Handles lens table data management
│   ├── visualize.js       # Visualizes the optical system
│   └── styles.css        # CSS styles for the application
├── package.json          # npm configuration file
└── README.md             # Project documentation
```

## Installation
1. Clone the repository:
   ```
   git clone <repository-url>
   ```
2. Navigate to the project directory:
   ```
   cd optical-system-visualizer
   ```
3. Install the dependencies:
   ```
   npm install
   ```

## Usage
1. Open `src/index.html` in a web browser to launch the application.
2. Use the interface to add, delete, and modify lens parameters in the table.
3. Visualize the optical system by interacting with the graphical representation generated by the application.

## Features
- Add and delete lens surfaces.
- Modify lens parameters such as radius, thickness, and material.
- Visual representation of the optical system based on the lens data.

## GitHub Pages (docs/) で公開する

このプロジェクトは静的サイトとして公開できます（サーバ不要）。

### 1) 公開用ファイルを `docs/` に生成

リポジトリ直下で:

```
node tools/build-pages-docs.mjs
```

- `docs/` は GitHub Pages が配信する“公開物”です。生成後は `docs/` をコミットしてください。
- 公開時は `https://<user>.github.io/<repo>/` のような **サブパス**で配信されます。相対パス参照が前提なので、`index.html` などで `/xxx` のような絶対パス参照は避けてください。

### 2) GitHub Pages を有効化

GitHub のリポジトリ画面で:

- Settings → Pages
- Source: Deploy from a branch
- Branch: `main`
- Folder: `/docs`

数分後に `https://<user>.github.io/<repo>/` でアクセスできます。

### 3) カスタムドメイン（例: co-opt.com）

Settings → Pages → Custom domain で `co-opt.com` を設定し、GitHub が表示する DNS 設定値（A/AAAA/CNAME）をドメインのDNSに反映してください。

### AI 機能について（重要）

- API key を公開サイトに埋め込むことはできません（誰でも取得できてしまいます）。
- 現状は「ユーザー各自が API key を入力」する方式で、キーはブラウザの `localStorage`（`ai_assistant_config`）に保存されます。
